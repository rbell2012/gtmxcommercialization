# Git/Change Logging Rule
When the user's full text is "Approved", agent must:
1. Summarize the changes made in the current session.
2. Identify the rationale/intent behind these changes.
3. Append a new entry to a file named `CHANGELOG.md` (or `COMMIT_LOG.md`) using the following format:

## Location - Front end Page/s the change effected
**Rationale:** [Why the change was made]
**Changes:** - [Detailed bullet point 1]
- [Detailed bullet point 2]
---

This command will be available in chat with /approved
