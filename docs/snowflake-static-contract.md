# Snowflake Static Metrics Contract

This defines the JSON contract for metrics files generated by the Snowflake sync job and served from `public/data`.

## Files (7 static JSON exports)

- `public/data/metrics_ops.json`
- `public/data/metrics_activity.json`
- `public/data/metrics_demos.json`
- `public/data/metrics_feedback.json`
- `public/data/metrics_chorus.json`
- `public/data/superhex.json`
- `public/data/metrics_tam.json`

**Not generated as static files:**

- **Calls / connects** — derived in the frontend from `metrics_activity` (same filters as Hex cells [36]/[37]).
- **Wins** — derived from `metrics_ops` via `isWinStage()` and enriched with `win_snapshots` in Supabase.

## Per-file column contract

### `metrics_ops.json`

- `id`
- `rep_name`
- `op_date`
- `op_created_date`
- `opportunity_name`
- `opportunity_stage`
- `win_stage_date`
- `opportunity_type`
- `opportunity_software_mrr`
- `account_prospecting_notes`
- `line_items`
- `account_name`
- `salesforce_accountid` (if present in warehouse export)

### `metrics_activity.json`

Columns required for dashboard **and** client-side calls/connects derivation:

- `id` (recommended for stable Supabase pagination)
- `rep_name`
- `activity_date`
- `salesforce_accountid`
- `activity_type`
- `activity_outcome`
- `subject`

### `metrics_demos.json`

- `rep_name`
- `demo_date`
- `account_name`
- `event_status`
- `demo_source`
- `subject`
- `salesforce_accountid`

### `metrics_feedback.json`

- `rep_name`
- `feedback_date`
- `account_name`
- `source`
- `feedback`

### `metrics_chorus.json`

- `id`
- `account_name`
- `salesforce_accountid`
- `comments`
- `chorus_link`
- `rep_name`
- `chorus_date`
- `rn` (row number from Hex dedupe; optional for UI)

### `superhex.json`

From the inlined notebook cell 29 export (Snowflake):

- `id` (MD5 of account + rep)
- `salesforce_accountid`
- `account_name`
- `rep_name`
- `total_activities`, `first_activity_date`, `last_activity_date`
- `total_calls`, `first_call_date`, `last_call_date`
- `total_connects`, `first_connect_date`
- `chorus_link`, `chorus_date`
- `total_demos`, `first_demo_date`
- `op_name`, `op_date`, `op_stage`
- `is_won`, `win_date`
- `feedback`, `feedback_date` — **NULL** until Sheet-backed feedback is in the warehouse (Cell 28 path dropped)
- `prospecting_notes` (from `ACCOUNT.account_prospecting_notes`)

(Additional columns may exist; the app tolerates extras.)

### `metrics_tam.json`

- `rep_name`
- `tam`

## Data types and serialization

- All files contain a top-level JSON array.
- Date/time values are serialized as ISO strings.
- Numeric fields remain numeric when possible.
- `null` values are preserved as `null`.
- Keys are snake_case to match existing frontend expectations.
